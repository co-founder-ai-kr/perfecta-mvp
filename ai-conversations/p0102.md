# 민순의 요청

```text
* 역할: 너는 AI 프롬프트 엔지니어링 및 의사소통 전문가야.
* 상황: 내 머리 속에 있는 단순한 지시를 AI가 잘 알아들을 수 있는 구체적인 구조(역할, 상황, 과업, 제약)로 바꾸고 싶어.
* 과업: 내 단순 지시를 구조화된 프롬프트로 업그레이드해 줘.
* 제약: 결과물은 복사해서 바로 쓸 수 있는 마크다운 코드 블록으로 출력해 줘.
* 입력: [나의 단순한 지시]
 (예시: 선물 추천 사이트 만들어 줘)
```

# 제미나이의 응답

네, 알겠습니다. 제시해주신 **예시(선물 추천 사이트 만들어 줘)**를 입력값으로 가정하고, 이를 AI가 완벽하게 이해하고 수행할 수 있는 **구조화된 프롬프트**로 변환해 드리겠습니다.

만약 다른 지시사항이 있다면, 언제든 편하게 말씀해 주세요.

***

```markdown
# Role: 시니어 풀스택 웹 개발자 및 프로덕트 매니저 (Senior Full-stack Developer & PM)

## 1. Situation (상황)
사용자는 선물 고르기에 어려움을 겪는 사람들을 돕기 위해 '선물 추천 웹사이트'를 구축하고자 합니다. 현재 아이디어 단계에 있으며, 이를 실현하기 위한 구체적인 기능 기획과 즉시 실행 가능한 프로토타입 코드가 필요한 상황입니다.

## 2. Task (과업)
다음 단계에 맞춰 선물 추천 사이트 구축을 위한 포괄적인 가이드를 작성해 주세요.

1.  **기능 기획 (Feature Planning)**:
    *   사용자가 대상(친구, 연인, 부모님 등), 기념일, 예산 범위를 선택할 수 있는 필터링 로직을 정의하세요.
    *   사용자 경험(UX)을 고려한 사이트의 핵심 페이지 구성을 나열하세요.
2.  **기술 스택 제안 (Tech Stack)**:
    *   빠른 MVP(최소 기능 제품) 제작을 위해 가장 효율적인 프론트엔드 및 백엔드 기술 스택을 추천하고 그 이유를 설명하세요.
3.  **프로토타입 코드 작성 (Coding)**:
    *   HTML, CSS, JavaScript를 사용하여 메인 페이지의 UI와 간단한 추천 로직이 작동하는 단일 파일(또는 분리된 코드 블록) 형태의 코드를 작성하세요.
    *   디자인은 모던하고 깔끔한 스타일(예: 카드 뉴스 형태의 상품 나열)을 적용하세요.

## 3. Constraints (제약)
*   **코드 품질**: 제공되는 코드는 복사하여 바로 브라우저에서 실행했을 때 작동해야 합니다.
*   **가독성**: 코드의 주요 부분에는 한글 주석을 달아 초보자도 이해할 수 있게 하세요.
*   **디자인**: CSS는 반응형으로 작성하여 모바일과 데스크톱 모두에서 깨지지 않도록 하세요.
*   **출력 형식**: 기획 내용은 마크다운 리스트로, 코드는 언어별 코드 블록으로 명확히 구분하여 출력하세요.
```
