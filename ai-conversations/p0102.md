# 민순의 요청

```text
* 역할: 너는 AI 프롬프트 엔지니어링 및 의사소통 전문가야.
* 상황: 내 머리 속에 있는 단순한 지시를 AI가 잘 알아들을 수 있는 구체적인 구조(역할, 상황, 과업, 제약)로 바꾸고 싶어.
* 과업: 내 단순 지시를 구조화된 프롬프트로 업그레이드해 줘.
* 제약: 결과물은 복사해서 바로 쓸 수 있는 마크다운 코드 블록으로 출력해 줘.
* 입력: [나의 단순한 지시]
 (예시: 선물 추천 사이트 만들어 줘)
```

# 제미나이의 응답

알겠습니다. 입력해주신 예시(**"선물 추천 사이트 만들어 줘"**)를 바탕으로, AI가 명확하게 이해하고 고품질의 답변을 내놓을 수 있도록 구조화된 프롬프트를 작성했습니다.

아래 코드를 복사해서 AI에게 입력해 보세요.

```markdown
# Role
너는 10년 차 시니어 풀스택 웹 개발자이자 UX/UI 기획 전문가야. 최신 웹 트렌드에 밝고, 사용자 경험(UX)을 최우선으로 고려하여 서비스를 설계하는 데 능숙해.

# Context
나는 현재 기념일, 성별, 연령대, 예산 등에 맞춰 적절한 선물을 추천해 주는 웹 사이트를 만들고 싶어. 하지만 구체적인 기능 명세나 기술적인 로드맵이 아직 잡히지 않은 상태야. 이 사이트는 사용자가 고민 없이 빠르게 선물을 고를 수 있도록 돕는 것이 목표야.

# Task
위 상황을 바탕으로 '선물 추천 사이트' 구축을 위한 구체적인 기획안을 작성해 줘. 포함해야 할 내용은 다음과 같아:
1. **핵심 기능 정의**: 사용자가 필수적으로 필요로 하는 기능 5가지 (예: 필터링, 찜하기 등)
2. **기술 스택 추천**: 프론트엔드, 백엔드, 데이터베이스 등 개발에 적합한 기술과 그 선정 이유
3. **데이터 구조**: 선물 데이터를 관리하기 위한 기본적인 데이터베이스 스키마(속성) 제안
4. **UI/UX 구조**: 메인 페이지와 상세 페이지의 와이어프레임(레이아웃) 텍스트 묘사

# Constraints
- 개발 지식이 부족한 사람도 이해할 수 있도록 전문 용어에는 간단한 설명을 덧붙여 줘.
- 모바일 환경에서의 사용성(Mobile First)을 최우선으로 고려한 설계를 해줘.
- 출력 형식은 가독성을 위해 마크다운의 헤더와 불렛 포인트를 활용해 정리해 줘.
```
