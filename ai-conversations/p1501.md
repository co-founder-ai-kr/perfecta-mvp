```text
* 역할: 너는 스타트업 전문 기록가이자 기술 문서 작성자야.
* 상황: 그동안의 대화 기록을 바탕으로 초보자도 따라 할 수 있는 가이드를 만들고 싶어.
* 과업: 
  1. '서비스 구축 플레이북'을 작성해 줘. 각 단계별 핵심 프롬프트와 해결했던 주요 오류를 포함해 줘.
  2. 다음 사업에도 쓸 수 있는 '1인 창업 스타터 키트'(HTML 템플릿, 구글 폼 코드)를 부록으로 만들어 줘.
* 제약: 
  - 단순히 나열하지 말고 '왜 이 프롬프트를 썼는지' 비즈니스 의도를 주석으로 달아 줘.
  - 배포 단계의 '드래그 앤 드롭 실패' 문제는 '주의 사항'으로 강조해 줘.
* 출력: 구글 독에 바로 붙여넣을 수 있는 마크다운(Markdown) 형식의 가이드.
* 입력: [그동안 축적된 주요 대화 로그 파일 첨부하기]
```

# 제미나이의 응답

첨부해주신 대화 로그(가상의 맥락)를 바탕으로, **[1인 창업가를 위한 AI 기반 MVP 구축 플레이북]**을 정리했습니다.

이 문서는 비개발자도 따라 할 수 있도록 **기획 → 개발 → 배포**의 흐름으로 구성되었으며, 각 단계에서 AI에게 어떤 질문을 던져야 비즈니스적으로 유의미한 결과를 얻을 수 있는지에 집중했습니다.

구글 독스(Google Docs)나 노션(Notion)에 바로 복사하여 사용할 수 있는 마크다운 형식입니다.

# 📘 1인 창업가를 위한 서비스 구축 플레이북 (Zero to One)

> **작성자:** 스타트업 전문 기록가 & 테크 라이터
> **버전:** 1.0
> **목표:** 코딩 지식이 없어도 AI를 활용해 하루 만에 랜딩 페이지를 만들고 고객 DB를 수집한다.

---

## 1단계: 기획 및 가설 설정 (Ideation)

무작정 코드를 짜는 것이 아니라, **'누구에게 무엇을 팔 것인가'**를 명확히 하는 단계입니다.

### 🎯 핵심 프롬프트
```markdown
**역할:** 너는 10년 차 린 스타트업(Lean Startup) PM이야.
**상황:** 나는 [반려동물 수제 간식 정기구독] 서비스를 준비 중이야.
**요청:** 
1. 이 서비스의 핵심 타겟 페르소나를 1명 정의해 줘.
2. 그 페르소나를 설득할 수 있는 '메인 카피(Head Copy)'와 '서브 카피'를 3가지 버전으로 제안해 줘.
3. 랜딩 페이지에 반드시 들어가야 할 필수 섹션 5가지를 순서대로 나열해 줘.
```

> **💡 비즈니스 의도 (Why?):**
> *   **페르소나 정의:** 모든 사람을 만족시키려다 아무도 만족시키지 못하는 상황을 방지합니다.
> *   **카피라이팅:** 개발보다 중요한 것은 '고객이 클릭하게 만드는 문구'입니다. AI에게 여러 버전을 요청해 A/B 테스트를 준비합니다.

---

## 2단계: UI/UX 디자인 및 코딩 (Development)

디자인 툴(Figma)을 건너뛰고, 바로 코드로 시각화하여 속도를 높입니다.

### 🎯 핵심 프롬프트
```markdown
**역할:** 너는 시니어 프론트엔드 개발자야.
**도구:** HTML, Tailwind CSS (CDN 방식), FontAwesome
**요청:** 
1. 위에서 기획한 내용을 바탕으로 반응형 랜딩 페이지 코드를 작성해 줘.
2. 디자인은 '모던하고 신뢰감을 주는' 톤앤매너(파란색 계열)로 해줘.
3. 상단에는 [로고, 네비게이션], 중간에는 [히어로 섹션, 특징 3가지, 고객 후기], 하단에는 [사전 예약 신청 폼, 푸터]를 배치해 줘.
4. 모든 코드는 하나의 'index.html' 파일에 합쳐서 줘.
```

> **💡 비즈니스 의도 (Why?):**
> *   **Tailwind CSS (CDN):** 복잡한 설치 과정 없이 바로 디자인을 적용하기 위해 선택했습니다.
> *   **단일 파일(One Page):** 파일 관리가 서툰 초보자가 파일 경로 문제로 헤매지 않도록 모든 리소스를 한 파일에 집중시킵니다.

### 🛠️ 주요 오류 해결: 이미지가 안 나올 때
*   **증상:** 코드를 실행했는데 이미지가 깨져서 나옴.
*   **해결책:** "이미지 부분은 `https://via.placeholder.com/400` 같은 임시 이미지가 아니라, Unsplash의 무료 이미지 URL을 랜덤으로 넣어서 수정해 줘."라고 재요청.

## 3단계: 데이터 수집 로직 연결 (Backend)

서버 비용 0원으로 고객 데이터를 수집하기 위해 **Google Sheets**를 데이터베이스로 활용합니다.

### 🎯 핵심 프롬프트
```markdown
**역할:** 풀스택 개발자
**목표:** HTML 폼(Form)에 입력된 데이터를 내 구글 스프레드시트에 자동으로 저장하고 싶어.
**요청:**
1. 구글 앱스 스크립트(Google Apps Script)에 붙여넣을 `doPost` 함수 코드를 작성해 줘.
2. HTML의 `<script>` 태그 안에 넣을 `fetch` 함수(데이터 전송 로직)를 작성해 줘.
3. 전송 성공 시 "신청이 완료되었습니다!"라는 알림창(alert)을 띄워 줘.
```

> **💡 비즈니스 의도 (Why?):**
> *   **구글 시트 연동:** 별도의 관리자 페이지를 개발할 필요 없이, 엑셀처럼 편하게 고객 명단을 관리할 수 있습니다. (초기 비용 절감)

### ⚠️ 주의: CORS 오류 (가장 많이 겪는 문제)
*   **증상:** "Network Error" 또는 콘솔창에 붉은색 CORS 에러 발생.
*   **해결책:** 구글 앱스 스크립트 배포 시 **'액세스 권한'을 반드시 '모든 사용자(Anyone)'로 설정**해야 합니다. (나만 보기 X)

## 4단계: 배포 및 출시 (Deployment)

내 컴퓨터에 있는 파일을 전 세계 사람들이 볼 수 있는 URL로 만듭니다. (Netlify 활용)

### 🎯 핵심 프롬프트
(이 단계는 프롬프트보다 실행 가이드가 중요합니다.)
1. "Netlify Drop" 사이트에 접속한다.
2. 작성한 `index.html`이 들어있는 **폴더 자체**를 드래그한다.

### 🚨 [주의 사항] 배포 실패: 드래그 앤 드롭이 안 돼요!
많은 초보자가 여기서 좌절합니다. 아래 내용을 반드시 확인하세요.

> **❌ 실패 원인:**
> 1. `index.html` **파일 하나만** 드래그했을 때 (반드시 **폴더**를 드래그해야 함).
> 2. 메인 파일 이름이 `index.html`이 아닐 때 (예: `home.html`, `main.html` 등은 인식 불가).
> 3. 폴더 안에 한글 파일명이 섞여 있을 때.
>
> **✅ 해결 방법:**
> 바탕화면에 `project`라는 영어 이름의 폴더를 만들고, 그 안에 `index.html` 파일을 넣은 뒤, **`project` 폴더를 통째로** Netlify 화면에 드래그하세요.

## 2. 구글 앱스 스크립트 코드 (Backend)
*구글 시트 > 확장 프로그램 > Apps Script에 붙여넣으세요.*

```javascript
var SHEET_NAME = "시트1"; // 시트 탭 이름 확인 필수
var SCRIPT_PROP = PropertiesService.getScriptProperties();

function doPost(e) {
  var lock = LockService.getScriptLock();
  lock.tryLock(10000);

  try {
    var doc = SpreadsheetApp.openById(SCRIPT_PROP.getProperty("key"));
    var sheet = doc.getSheetByName(SHEET_NAME);

    var headers = sheet.getRange(1, 1, 1, sheet.getLastColumn()).getValues()[0];
    var nextRow = sheet.getLastRow() + 1;

    var newRow = headers.map(function(header) {
      return header === 'timestamp' ? new Date() : e.parameter[header];
    });

    sheet.getRange(nextRow, 1, 1, newRow.length).setValues([newRow]);

    return ContentService
      .createTextOutput(JSON.stringify({ 'result': 'success', 'row': nextRow }))
      .setMimeType(ContentService.MimeType.JSON);
  }

  catch (e) {
    return ContentService
      .createTextOutput(JSON.stringify({ 'result': 'error', 'error': e }))
      .setMimeType(ContentService.MimeType.JSON);
  }

  finally {
    lock.releaseLock();
  }
}

function setup() {
  var doc = SpreadsheetApp.getActiveSpreadsheet();
  SCRIPT_PROP.setProperty("key", doc.getId());
}
```
> **💡 설정 팁:**
> 1. 코드를 붙여넣고 `setup` 함수를 한 번 실행(Run)하여 권한을 허용하세요.
> 2. [배포] > [새 배포] > 유형: 웹 앱 > **액세스 권한: 모든 사용자(Anyone)** 로 설정 후 배포하세요.
> 3. 생성된 URL을 위 HTML 코드의 `YOUR_GOOGLE_SCRIPT_URL_HERE` 부분에 넣으세요.
