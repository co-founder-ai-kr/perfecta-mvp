# 민순의 요청

```text
* 역할: 너는 스타트업 전문 기록가이자 기술 문서 작성자야.
* 상황: 그 동안의 대화 기록을 바탕으로 누구나 따라 할 수 있는 가이드를 만들고 싶어.
* 과업: 
  1. '서비스 구축 플레이북'을 작성해 줘. 각 단계별 핵심 프롬프트와 해결했던 주요 오류를 포함해 줘.
  2. 다음 사업에도 쓸 수 있는 '1인 창업 스타터 키트'(HTML 템플릿, 구글 폼 코드)를 부록으로 만들어 줘.
* 제약: 
  - 단순히 나열하지 말고 '왜 이 프롬프트를 썼는지' 비즈니스 의도를 주석으로 달아 줘.
  - 배포 단계의 '드래그 앤 드롭 실패' 문제는 '주의 사항'으로 강조해 줘.
* 출력: 구글 독에 바로 붙여넣을 수 있는 마크다운(Markdown) 형식의 가이드.
* 입력: [대화 로그 파일]
 (예시: 기획부터 배포까지의 제미나이 채팅 내역 전체)
```

# 제미나이의 응답

제공해주신 대화 로그(가상의 시나리오)를 바탕으로, 1인 창업가가 AI를 활용해 서비스를 구축할 수 있는 **[AI 기반 1인 창업 서비스 구축 플레이북]**을 정리했습니다.

이 문서는 구글 닥스(Google Docs)나 노션(Notion)에 바로 붙여넣어 사용할 수 있도록 마크다운(Markdown) 형식으로 작성되었습니다.

# 📘 AI 기반 1인 창업 서비스 구축 플레이북 (Ver 1.0)

**작성자:** 스타트업 전문 기록가  
**목적:** 비개발자 1인 창업가가 AI와 협업하여 MVP(최소 기능 제품)를 기획, 개발, 배포하는 전 과정 가이드  
**대상:** 아이디어는 있지만 코딩 지식이 부족한 예비 창업가

---

## 🚀 Phase 1: 기획 및 아이디어 구체화 (Blueprint)

개발을 시작하기 전, AI에게 명확한 '비즈니스 요구사항'을 주입하는 단계입니다.

### 1. 페르소나 설정 및 기능 정의
단순히 "웹사이트 만들어줘"라고 하면 일반적인 결과물이 나옵니다. 구체적인 비즈니스 목표를 설정해야 합니다.

*   **사용 프롬프트:**
    ```markdown
    너는 지금부터 10년 차 시니어 PM이자 UX 디자이너야. 
    나는 [타겟 고객: 예 - 30대 직장인]을 위한 [서비스 주제: 예 - 뉴스레터 구독 서비스]를 만들고 싶어.
    이 서비스의 핵심 가치는 [가치: 예 - 매일 아침 3분 만에 트렌드 파악]이야.
    
    1. 이 서비스에 꼭 필요한 핵심 기능 3가지만 정의해 줘. (MVP 기준)
    2. 메인 랜딩 페이지에 들어갈 카피라이팅(헤드라인, 서브텍스트, CTA 버튼)을 작성해 줘.
    ```
    > **💡 Business Insight:**  
    > AI에게 '시니어 PM'이라는 역할을 부여하여 기획의 퀄리티를 높입니다. MVP(최소 기능 제품)로 범위를 한정해야 개발 기간이 늘어지는 것을 방지할 수 있습니다.

### 2. 와이어프레임(구조) 설계
디자인 툴을 켜기 전, 텍스트로 구조를 잡습니다.

*   **사용 프롬프트:**
    ```markdown
    위에서 정의한 내용을 바탕으로 랜딩 페이지의 HTML 구조(Structure)를 텍스트로 그려줘.
    헤더, 히어로 섹션, 특징 소개, 소셜 증거(후기), 푸터 순서로 배치하고 각 섹션에 들어갈 내용을 요약해 줘.
    ```

## 💻 Phase 2: 코드 생성 및 구현 (Construction)

실제 코드를 작성하는 단계입니다. 복잡한 프레임워크 대신 수정이 쉬운 HTML/CSS/JS를 사용합니다.

### 1. 기본 골격 및 디자인 생성
*   **사용 프롬프트:**
    ```markdown
    이제 실제 코드를 작성해 줘.
    - 기술 스택: HTML5, Tailwind CSS (CDN 방식), Vanilla JS
    - 디자인 스타일: 토스(Toss)나 에어비앤비처럼 깔끔하고 모던한 스타일, 폰트는 'Pretendard' 적용.
    - 반응형: 모바일에서도 완벽하게 보여야 함.
    
    하나의 index.html 파일 안에 모든 스타일과 스크립트를 포함해서 작성해 줘.
    ```
    > **💡 Business Insight:**  
    > `Tailwind CSS (CDN)`을 요청한 이유는 별도의 설치 과정(npm install 등) 없이 파일 하나로 디자인을 수정하기 가장 쉽기 때문입니다. '하나의 파일'로 요청하면 파일 관리 실수를 줄일 수 있습니다.

### 2. 주요 오류 해결 (Troubleshooting)
개발 중 가장 많이 겪는 문제와 해결책입니다.

*   **오류 상황:** 이미지가 깨져서 나오거나 레이아웃이 틀어짐.
*   **해결 프롬프트:**
    ```markdown
    "히어로 섹션의 이미지가 모바일에서 너무 크게 나와. 그리고 버튼을 눌렀을 때 반응이 없어.
    1. 이미지는 Unsplash의 랜덤 비즈니스 이미지 URL로 대체해 줘.
    2. 버튼 클릭 시 '준비 중입니다'라는 알림창(alert)이 뜨도록 수정해 줘."
    ```

## ⚠️ Phase 3: 배포 및 운영 (Launch)

서버 비용 없이 무료로 서비스를 런칭하는 단계입니다. (Netlify 활용)

### 1. 배포 준비 (폴더 정리)
가장 중요한 단계입니다. 많은 초보자가 여기서 실패합니다.

*   **체크리스트:**
    1.  바탕화면에 `my-startup` 폴더 생성.
    2.  AI가 짜준 코드를 메모장에 붙여넣고 `index.html`로 저장.
    3.  **주의:** 파일명이 반드시 `index.html`이어야 합니다. (`home.html` (X), `index.html.txt` (X))

### 2. Netlify 드래그 앤 드롭 배포
*   **절차:** Netlify 로그인 -> 'Sites' 메뉴 -> 'Add new site' -> 'Deploy manually'
*   **🚨 주의 사항: 드래그 앤 드롭 실패 (Page Not Found)**
    > **증상:** 배포는 됐는데 접속하면 "Page Not Found"가 뜸.  
    > **원인:** 폴더 구조 문제. `index.html` 파일만 드래그하거나, `index.html`이 이중 폴더(`folder/folder/index.html`) 안에 있을 때 발생함.  
    > **해결책:** 
    > 1. `index.html`이 들어있는 **최상위 폴더 자체**를 드래그해야 함.
    > 2. 폴더를 열었을 때 바로 `index.html`이 보여야 함.

## 🎁 부록: 1인 창업 스타터 키트

다음 사업 아이템 검증 시 바로 복사해서 쓸 수 있는 템플릿입니다.

### 1. 만능 랜딩 페이지 템플릿 (HTML + Tailwind CSS)
이 코드를 `index.html`로 저장하면 즉시 사용 가능합니다.

```html
<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>서비스 이름 - 한 줄 소개</title>
    <script src="https://cdn.tailwindcss.com"></script>
    <link rel="stylesheet" as="style" crossorigin href="https://cdn.jsdelivr.net/gh/orioncactus/pretendard@v1.3.9/dist/web/static/pretendard.min.css" />
    <style>
        body { font-family: "Pretendard Variable", Pretendard, -apple-system, BlinkMacSystemFont, system-ui, Roboto, sans-serif; }
    </style>
</head>
<body class="bg-gray-50 text-gray-900">

    <!-- 히어로 섹션 -->
    <section class="flex flex-col items-center justify-center min-h-screen px-4 text-center">
        <span class="bg-blue-100 text-blue-800 text-xs font-semibold px-2.5 py-0.5 rounded mb-4">Beta Service</span>
        <h1 class="text-4xl font-bold tracking-tight sm:text-6xl mb-6">
            고객의 문제를 해결하는<br><span class="text-blue-600">핵심 가치 한 줄</span>
        </h1>
        <p class="text-lg text-gray-600 mb-8 max-w-2xl">
            여기에 서비스에 대한 구체적인 설명을 적으세요. 고객이 얻을 수 있는 이득을 중심으로 작성하는 것이 좋습니다.
        </p>
        
        <!-- 구글 폼 연동 영역 -->
        <div class="w-full max-w-md bg-white p-6 rounded-xl shadow-lg">
            <h3 class="text-lg font-semibold mb-4">사전 예약하고 혜택 받기</h3>
            <form id="submit-form" class="space-y-4">
                <!-- 아래 name 속성값은 구글 폼에서 가져와야 함 -->
                <input type="email" name="entry.YOUR_EMAIL_ID" placeholder="이메일 주소를 입력하세요" required 
                       class="w-full px-4 py-3 border border-gray-300 rounded-lg focus:ring-2 focus:ring-blue-500 focus:outline-none">
                <button type="submit" class="w-full bg-blue-600 text-white font-bold py-3 rounded-lg hover:bg-blue-700 transition">
                    무료로 시작하기
                </button>
            </form>
            <p class="text-xs text-gray-400 mt-4">스팸은 보내지 않습니다. 언제든 구독 취소 가능합니다.</p>
        </div>
    </section>

    <!-- 자바스크립트 (구글 폼 연동용) -->
    <script>
        const form = document.getElementById('submit-form');
        // 구글 폼의 'action' URL을 아래에 넣으세요
        const GOOGLE_FORM_URL = 'https://docs.google.com/forms/u/0/d/e/YOUR_FORM_ID/formResponse';

        form.addEventListener('submit', async (e) => {
            e.preventDefault();
            const formData = new FormData(form);
            
            try {
                await fetch(GOOGLE_FORM_URL, {
                    method: 'POST',
                    mode: 'no-cors', // 중요: 구글 폼 전송 시 필수
                    body: formData
                });
                alert('성공적으로 등록되었습니다! 곧 연락드리겠습니다.');
                form.reset();
            } catch (error) {
                alert('오류가 발생했습니다. 다시 시도해 주세요.');
            }
        });
    </script>
</body>
</html>
```

### 2. 구글 폼(Google Forms) 연동 가이드 (백엔드 대체)
서버 개발 없이 고객 DB를 쌓는 방법입니다.

1.  **구글 폼 생성:** 이메일 입력 질문 하나만 만들고 저장.
2.  **사전 채워진 링크 가져오기:**
    *   구글 폼 우측 상단 점 3개 클릭 -> '미리 채워진 링크 가져오기(Get pre-filled link)'
    *   이메일에 아무거나 입력 후 '링크 받기' 클릭.
    *   링크를 복사해 메모장에 붙여넣기.
3.  **코드 수정:**
    *   링크에서 `entry.123456789` 처럼 보이는 부분을 찾아 위 HTML 코드의 `name="entry.YOUR_EMAIL_ID"` 부분에 교체.
    *   구글 폼의 주소(`.../formResponse`로 끝나는 주소)를 `GOOGLE_FORM_URL` 변수에 넣기.

**기록가 코멘트:**
> 이 플레이북은 기술적인 깊이보다는 **'실행 속도'**에 초점을 맞췄습니다. 완벽한 코드가 아니더라도, 고객을 만나는 것이 가장 중요합니다. 이 가이드가 당신의 첫 번째 서비스 런칭에 도움이 되기를 바랍니다.
