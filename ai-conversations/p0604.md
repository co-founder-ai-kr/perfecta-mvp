# 제미나이의 응답

반갑습니다. 린 스타트업 코치입니다.

지금 겪고 계신 **'기능 비만(Feature Creep)'** 상태는 초기 창업가가 겪는 가장 흔하고 위험한 함정입니다. 완벽하게 만들고 싶은 욕심이 오히려 실행을 가로막고 있는 것이죠.

포기하지 마세요. 우리가 지금 해야 할 일은 프로젝트 포기가 아니라, **'군살 제거(지방 흡입)'**입니다.

**작성하신 PRD(기획서) 내용을 이 채팅창에 붙여넣어 주세요.**

내용을 주시면, 제가 다음과 같은 **[사고 과정]**을 거쳐 당신의 서비스를 **'개발 기간 3일 이내'**의 초경량 MVP로 재설계해 드리겠습니다.

---

### 🛠️ 코치의 MVP 축소 사고 과정 (미리보기)

제가 당신의 기획서를 분석할 때 사용할 기준은 다음과 같습니다.

1.  **핵심 가설(Core Hypothesis) 단 하나만 남기기:**
    *   "이 기능이 없으면 우리 서비스의 존재 이유가 사라지는가?" → NO라면 삭제.
2.  **자동화 대신 수동화 (Concierge MVP):**
    *   "개발로 자동화하려는 기능을 내가 직접 손으로(엑셀, 카톡 등) 처리할 수 있는가?" → YES라면 개발 삭제.
3.  **확장성 무시 (Do things that don't scale):**
    *   회원가입, 로그인, 마이페이지, 관리자 페이지 등은 초기 10명의 고객에게는 필요 없습니다. → 삭제.
4.  **3일 컷 (The 3-Day Rule):**
    *   이 기능을 구현하는 데 4시간 이상 걸릴 것 같은가? → 삭제하거나 노코드로 대체.

---

### 📋 (예시) 출력 형식

기획서를 주시면 아래와 같은 표로 정리해 드립니다.

| 구분 | 기능명 | 처리 (삭제/유지) | 코치의 조언 (삭제 이유 및 대안) |
| :--- | :--- | :--- | :--- |
| **핵심** | **메인 매칭 기능** | **✅ 유지** | 이것이 검증하려는 핵심 가치입니다. 단, 알고리즘 대신 구글 폼으로 받으세요. |
| 부가 | 회원가입/로그인 | ❌ 삭제 | 구글 폼에는 로그인이 필요 없습니다. 이메일 주소만 받으세요. |
| 부가 | 결제 시스템 PG 연동 | ❌ 삭제 | 계좌이체로 받으세요. PG사 심사만 2주 걸립니다. |
| 부가 | 마이페이지/이력 | ❌ 삭제 | 고객에게 결과는 이메일이나 카톡으로 쏴주면 됩니다. |
