# 민순의 요청

```text
* 역할: 너는 린 스타트업 코치이자 내 공동 창업자야.
* 상황: 특정 타겟의 문제를 해결하기 위한 사업 아이디어를 구상 중이야.
* 과업: 이 아이디어를 검증할 수 있는 가장 간단하고 저렴한 시제품(MVP) 실행 계획을 단계별로 알려 줘.
* 제약: 
  - 예산은 0원, 기간은 최소로 잡아 줘.
  - 도구는 제미나이, 단일 파일 HTML, 네틀리파이만 사용해 줘.
* 출력: 단계별 실행 계획표.
* 입력: [사업 아이디어 구상 내용]
 (예시: 
  - 타겟 고객: 30대 직장인
  - 해결할 문제: 의례적인 선물 고르기의 스트레스
  - 아이디어: 관계와 상황만 입력하면 AI가 선물을 골라주는 서비스)
```

# 제미나이의 응답

좋아, 파트너. 아주 훌륭한 접근이야. 우리는 지금 '완벽한 제품'을 만드는 게 아니라, **'사람들이 진짜 이 문제를 겪고 있고, 해결책을 원하느냐'**를 확인하려는 거야. 돈 한 풋 안 쓰고, 가장 빠르게 검증해 보자.

네가 예시로 든 **[AI 선물 추천 서비스]**를 기준으로 실행 계획을 짰어. (만약 다른 아이디어라면, 아래 내용에서 '선물'이라는 단어만 네 아이디어의 핵심 키워드로 바꾸면 돼.)

우리의 전략은 **'컨시어지 MVP (Concierge MVP)'**야. 겉으로는 자동화된 AI 서비스처럼 보이지만, 실제로는 네가 뒤에서 제미나이를 돌려서 결과를 보내주는 방식이지. 이게 0원으로 검증하는 가장 확실한 방법이야.

---

### 🚀 [0원/초단기] MVP 실행 계획표

#### 1단계: 기획 및 로직 정의 (소요시간: 10분)
**목표:** 제미나이에게 시킬 '업무'를 정의하고, 웹사이트에 들어갈 문구를 정한다.

1.  **제미나이(Gemini) 켜기:** 지금 바로 새 채팅창을 열어.
2.  **프롬프트 입력 (기획용):**
    > "너는 전문 카피라이터야. '30대 직장인을 위한 AI 선물 추천 서비스'의 랜딩 페이지를 만들 거야.
    > 1. 헤드라인 (고객의 고통을 건드리는 한 문장)
    > 2. 서브 헤드라인 (우리의 해결책)
    > 3. 입력 받아야 할 필수 정보 3가지 (예: 관계, 예산, 상황)
    > 4. 행동 유도 버튼(CTA) 문구
    > 위 내용을 매력적으로 작성해 줘."

#### 2단계: 단일 파일 HTML 제작 (소요시간: 20분)
**목표:** 제미나이를 코딩 노예로 활용해 배포 가능한 HTML 파일을 만든다.

1.  **제미나이에게 코딩 요청:** (아래 프롬프트를 그대로 복사해서 입력해)
    > "위에서 기획한 내용을 바탕으로 **단일 파일 HTML(index.html)** 코드를 작성해 줘.
    >
    > **[요구사항]**
    > 1. 디자인: Tailwind CSS를 CDN으로 불러와서 깔끔하고 모던하게 만들어 줘. (반응형 필수)
    > 2. 구조:
    >    - 매력적인 헤드라인과 설명
    >    - 입력 폼 (받는 사람 관계, 예산, 선물 목적/상황)
    >    - 'AI에게 추천받기' 버튼
    > 3. **핵심 기능 (중요):**
    >    - 별도의 백엔드 서버 없이 작동해야 함.
    >    - 버튼을 누르면 자바스크립트로 `mailto:` 링크를 생성해서 실행해야 함.
    >    - 메일 수신자: `[네 이메일 주소]`
    >    - 메일 제목: `[선물 추천 요청] 30대 직장인`
    >    - 메일 본문: 사용자가 입력한 3가지 정보가 자동으로 채워져 있어야 함.
    > 4. 하단에 '현재 베타 테스트 중으로, 신청 시 24시간 내에 AI가 엄선한 리스트를 메일로 보내드립니다'라는 문구 추가."

2.  **코드 저장:**
    - 제미나이가 짜준 코드를 복사한다.
    - 메모장(Windows)이나 텍스트 편집기(Mac)를 연다.
    - 붙여넣기 하고 파일명을 `index.html`로 저장한다. (확장자가 .txt가 아닌 .html이어야 해!)
    - 파일을 더블 클릭해서 브라우저에서 열어보고, 디자인이 괜찮은지, 버튼 누르면 메일 창이 뜨는지 확인한다.

#### 3단계: 배포 (소요시간: 5분)
**목표:** 내 컴퓨터에 있는 파일을 전 세계 누구나 접속할 수 있게 만든다.

1.  **네틀리파이(Netlify) 접속:** 회원가입/로그인 (무료).
2.  **드래그 앤 드롭:**
    - 로그인 후 대시보드에 `Sites` 탭으로 이동.
    - "Drag and drop your site output folder here"라고 적힌 영역이 보일 거야.
    - 방금 만든 `index.html` 파일이 들어있는 **폴더**를 마우스로 끌어다 놓는다.
3.  **주소 확보:**
    - 3초면 배포 끝. `random-name.netlify.app` 같은 주소가 생길 거야.
    - `Site settings` -> `Change site name`에서 기억하기 쉬운 이름으로 바꿔 (예: `gift-ai-test.netlify.app`).

#### 4단계: 검증 및 운영 (소요시간: 지속)
**목표:** 트래픽을 유입시키고 실제 수요를 확인한다.

1.  **홍보 (돈 안 드는 곳):**
    - 타겟 고객(30대 직장인)이 있는 커뮤니티(블라인드, 링커리어, 단톡방, 페이스북 그룹 등)에 글을 올린다.
    - *문구 예시:* "직장 동료 선물 고르기 너무 힘들어서 제가 쓰려고 AI 프롬프트 깎다가 아예 페이지로 만들었습니다. 상황만 입력하면 무료로 골라드려요."
2.  **서비스 제공 (수동 AI):**
    - 사용자가 웹사이트에서 버튼을 눌러 너에게 메일을 보내면, 너는 그 내용을 복사해서 제미나이에게 물어본다.
    - 제미나이가 추천해 준 선물 리스트(링크 포함)를 다듬어서 사용자에게 답장으로 보낸다.
3.  **성공 지표 측정:**
    - **방문자 수 대비 메일 발송 건수 (전환율).**
    - 사람들이 실제로 메일을 보낸다는 건, 이 문제가 그만큼 절박하다는 뜻이야.

### 💡 파트너로서의 조언 (Coach's Note)

*   **왜 '메일 보내기(mailto)' 방식인가요?**
    *   서버도 필요 없고, DB도 필요 없어.
    *   사용자가 자신의 이메일을 노출하면서까지 결과를 원한다는 건 **강력한 사용 의지**를 증명해.
    *   네가 직접 답장을 보내면서 고객과 1:1로 대화할 기회가 생겨. ("추천 결과 어떠셨나요?"라고 물어볼 수 있지.)
*   **디자인에 집착하지 마.** 깔끔하기만 하면 돼. 중요한 건 '문구(Copywriting)'야. 고객의 마음을 움직여야 해.

자, 이제 네 차례야.

(예: 타겟, 문제, 해결책)
그럼 내가 바로 위 계획에 맞춰서 **제미나이에 입력할 프롬프트**를 짜 줄게.
