# 민순의 요청

```text
* 역할: 너는 웹 프론트엔드 호환성 점검 전문가야.
* 상황: 배포한 웹 사이트의 일부 기능이 작동하지 않는데, 외부에서 가져온 코드(CDN) 문제인 것 같아.
* 과업: HTML 파일 상단의 `<script>` 태그들을 확인해서 너무 오래된 버전이거나 서로 충돌할 가능성이 있는지 점검해 줘.
* 제약: 초보자 눈높이에서 설명해 줘.
* 출력: 호환성 점검 결과
* 입력: [index.html 코드 전체 붙여넣기]
```

# 제미나이의 응답

안녕하세요! 웹 프론트엔드 호환성 점검 전문가입니다. 🫡

보내주신 내용을 보니, **아직 분석할 `index.html` 코드가 입력되지 않았습니다.**
입력란에 `[index.html 코드 전체 붙여넣기]`라고만 적혀 있고 실제 코드가 없네요!

문제가 되는 **HTML 파일의 내용을 답장으로 붙여넣어 주시면**, 제가 바로 돋보기를 들고 점검해 드리겠습니다.

코드를 주시면 다음과 같은 기준으로 **초보자도 이해하기 쉽게** 딱 집어 드릴게요.

1.  **👴 할아버지 버전 찾기:** 너무 오래되어서 요즘 기능들과 말이 안 통하는 버전인지 확인합니다. (예: 10년 전 jQuery 등)
2.  **🥊 코드끼리의 싸움(충돌):** 같이 쓰면 서로 방해하는 라이브러리가 있는지 확인합니다. (예: 똑같은 도구를 두 번 불러오거나, 서로 안 맞는 버전끼리 섞어 쓰는 경우)
3.  **👕 순서 엉킴:** 옷을 입을 때 속옷을 먼저 입어야 하듯, 코드도 불러오는 순서가 중요한데 그게 꼬였는지 확인합니다.
4.  **🔗 주소 오류:** 코드를 가져오는 창고(CDN) 주소가 문을 닫았거나 잘못되었는지 확인합니다.

**지금 바로 코드를 붙여넣어 주세요!** 꼼꼼하게 봐드리겠습니다.
