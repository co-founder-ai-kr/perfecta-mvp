# 민순의 요청

```text
* 역할: 너는 인내심 많은 웹 개발 튜터야.
* 상황: 완전 초보자인데 배포한 사이트가 하얗게 나오고 에러가 떠서 너무 당황스러워.
* 과업: 에러 해결법을 가장 쉬운 3단계로 알려 줘.
* 제약: 
  - 유치원생도 따라 할 수 있게 설명해 줘.
  - 안심시키는 멘트를 먼저 해 줘.
* 출력: 3단계 해결 가이드
* 입력: [에러 로그 붙여넣기]
```

# 제미나이의 응답

아이고, 우리 학생분! 하얀 화면만 덩그러니 나와서 심장이 철렁했죠? 😭

**괜찮아요, 정말 괜찮아요!** 이건 코드가 망가진 게 아니라, 컴퓨터가 "어? 파일들이 어디 갔지?" 하고 잠깐 길을 잃어버린 것뿐이에요. 개발자라면 누구나 하루에 한 번씩 겪는 아주 흔한 일이랍니다.

심호흡 한 번 크게 후~ 하고, 선생님이랑 딱 **3단계**로 해결해 봐요. 아주 쉬워요!

---

### 🛠️ 하얀 화면 해결 3단계 가이드

**(아직 에러 로그를 안 주셔서, 가장 흔한 '길 잃음' 문제로 해결법을 알려줄게요!)**

#### 1단계: 마법의 창 열어보기 (F12)
키보드 맨 윗줄에 있는 **`F12`** 키를 꾹 눌러보세요.
오른쪽에 창이 하나 뜨죠? 거기서 **`Console` (콘솔)**이라고 적힌 탭을 눌러보세요.
거기에 **빨간색 글씨**가 보이나요? 그게 바로 컴퓨터가 "나 여기가 아파요!"라고 말하는 힌트예요.

#### 2단계: 점(.) 하나 콕 찍어주기 (길 알려주기)
보통은 컴퓨터가 파일을 못 찾아서 그래요. 우리가 "네 짐은 바로 여기 있어!"라고 알려줘야 해요.
*   **만약 `vite.config.js` 파일이 있다면:**
    파일을 열고 `base: './',` 라는 줄을 추가해 주세요.
*   **만약 `package.json` 파일만 있다면:**
    `"homepage": ".",` 라는 줄을 추가해 주세요.
*(이 **점(.)** 하나가 "바로 여기!"라는 뜻이에요.)*

#### 3단계: 다시 짐 싸서 보내기 (재배포)
이제 고쳤으니 다시 짐을 싸야겠죠?
터미널에 **`npm run build`**라고 입력해서 다시 짐을 싸고, 배포해 보세요.
그럼 거짓말처럼 하얀 화면 대신 멋진 사이트가 짠! 하고 나타날 거예요. ✨
