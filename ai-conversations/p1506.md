# 제미나이의 응답

반갑습니다. IT 기업 M&A 기술 실사 전문가이자 테크니컬 라이터로서, 귀하의 소중한 자동화 시스템이 구매자에게 안전하고 완벽하게 전달될 수 있도록 가이드를 작성해 드립니다.

기술을 잘 모르는 구매자에게 가장 중요한 것은 **'완벽한 소유권의 분리'**와 **'쉬운 사용법'**입니다. 이를 위해 먼저 개념을 명확히 잡고, 실무적인 절차를 안내해 드리겠습니다.

---

### 💡 핵심 개념: '공유' vs '소유권 이전' vs '사본 생성'

구매자에게 이 개념을 먼저 설명해야 오해가 없습니다. 우리는 **'가게(시스템)'**를 넘기는 상황입니다.

1.  **공유 (Sharing):**
    *   **비유:** 내 가게의 **'복사한 열쇠'**만 건네주는 것입니다.
    *   **위험:** 집주인(판매자)이 언제든 자물쇠를 바꾸면 구매자는 못 들어옵니다. 판매자가 실수로 파일을 지우면 구매자의 시스템도 사라집니다. **절대 비추천합니다.**
2.  **소유권 이전 (Transfer Ownership):**
    *   **비유:** 가게의 **'명의(등기)'**를 구매자 이름으로 바꾸는 것입니다.
    *   **제약:** 구글 개인 계정(@gmail.com)에서 만든 파일은 기업 계정(Google Workspace)이나 다른 도메인으로 소유권 이전이 **불가능한 경우**가 많습니다.
3.  **사본 생성 (Make a Copy) - ✅ 추천 방식:**
    *   **비유:** 내 가게와 똑같은 인테리어와 설비를 갖춘 **'새 가게'를 옆에 지어주고, 새 열쇠를 주는 것**입니다.
    *   **장점:** 판매자의 계정이 삭제되어도 구매자의 시스템은 안전합니다. 가장 확실한 인계 방법입니다.

---

### 📋 Part 1. 소유권 이전 및 전달 절차 체크리스트
*(판매자와 구매자가 함께 진행하거나, 판매자가 대행 후 계정을 넘기는 방식)*

이 체크리스트는 **'사본 생성'** 방식을 기준으로 작성되었습니다. 구글 시스템 특성상 가장 안전한 방법입니다.

#### 1단계: 사전 준비 (판매자)
- [ ] **코드 내 하드코딩 제거:** 스크립트 안에 내 개인 이메일 주소나, 특정 구글 시트의 고유 ID(URL에 있는 긴 문자열)가 박혀 있다면 지우거나 주석 처리했는지 확인하세요.
- [ ] **외부 서비스 계정 정리:** Netlify 등 외부 서비스는 이메일/비번을 변경하여 계정 자체를 넘길 준비를 합니다. (가능하다면 구매자 전용 이메일을 하나 만들어 가입 정보를 변경해 두세요.)

#### 2단계: 시스템 복제 및 전달 (가장 중요)
- [ ] **구글 시트 사본 만들기:**
    1.  구매자에게 구글 시트 '뷰어' 권한으로 공유합니다.
    2.  구매자가 본인 구글 계정으로 로그인한 상태에서 `파일` > `사본 만들기`를 클릭하게 합니다.
    3.  **결과:** 이제 그 파일은 100% 구매자의 것입니다. (판매자와 연결 끊김)
- [ ] **앱스 스크립트 확인:**
    *   구글 시트에 스크립트가 내장(Container-bound)된 경우, 시트를 복사하면 스크립트도 자동으로 따라옵니다. 별도로 옮길 필요 없습니다.
- [ ] **Netlify 계정 인계:**
    *   ID/PW를 전달하고, 구매자가 로그인 후 즉시 비밀번호와 복구 이메일을 변경하도록 안내합니다.

#### 3단계: 연결 재설정 (구매자 환경 세팅)
*사본을 만들면 '자동화 타이머(트리거)'는 복사되지 않습니다. 다시 켜줘야 합니다.*
- [ ] **스크립트 권한 승인:** 구매자가 처음 버튼을 누르거나 기능을 실행할 때 뜨는 '권한 요청' 팝업을 승인하도록 안내합니다. (매뉴얼 참조)
- [ ] **트리거(자동화) 재설정:**
    *   `확장 프로그램` > `Apps Script` > 왼쪽 시계 아이콘(트리거) > `트리거 추가`
    *   기존에 설정했던 시간(예: 매일 오전 9시 실행)과 똑같이 설정해 줍니다.

#### 4단계: 최종 검증
- [ ] 판매자의 원본 파일 이름을 변경하거나 삭제해 봅니다.
- [ ] 구매자의 시스템이 여전히 잘 작동하는지 확인합니다. (작동한다면 완벽히 분리된 것입니다.)

# [시스템 이름] 운영 및 관리 매뉴얼

본 매뉴얼은 구글 시트 및 자동화 시스템을 기술적 지식이 없는 관리자도 안전하게 운영할 수 있도록 작성되었습니다.

## 1. 시스템 개요
이 시스템은 **[입력된 데이터]**를 바탕으로 **[자동화된 작업]**을 수행하여 **[최종 결과물/Netlify 웹사이트]**에 반영합니다.

*   **구성 요소:**
    1.  **메인 데이터 시트:** 데이터가 저장되는 엑셀 같은 화면입니다.
    2.  **자동화 엔진 (Apps Script):** 시트 뒤에서 작동하는 로봇입니다.
    3.  **웹 뷰어 (Netlify):** 결과물을 보여주는 웹사이트입니다.

## 2. 데이터 수정 방법 (주의 사항)

시스템이 고장 나지 않게 하려면 **'건드려도 되는 곳'**과 **'안 되는 곳'**을 구분해야 합니다.

*   **✅ 수정 가능 영역 (흰색 배경):**
    *   데이터를 입력하거나 수정하는 칸입니다. 자유롭게 내용을 바꾸셔도 됩니다.
    *   행(가로줄)을 추가하여 새로운 데이터를 넣어도 됩니다.
*   **⛔ 수정 금지 영역 (회색 배경 또는 수식 칸):**
    *   자동으로 계산되거나 스크립트가 값을 뱉어내는 곳입니다.
    *   이곳을 지우거나 손으로 직접 입력하면 자동화가 끊길 수 있습니다.
    *   *실수로 지웠다면?* 즉시 `Ctrl + Z` (실행 취소)를 누르세요.

## 3. 최초 사용 시 필수 설정 (권한 승인)

처음 시스템을 실행하면 구글에서 **"확인되지 않은 앱"**이라는 무서운 경고창을 띄웁니다. 이는 바이러스가 아니라, 구글이 직접 만든 프로그램이 아니라는 뜻이니 안심하세요.

**[승인 절차]**
1.  `권한 검토` 버튼 클릭
2.  계정 선택
3.  "Google에서 확인하지 않은 앱" 화면이 나오면 왼쪽 하단 **`고급(Advanced)`** 클릭
4.  맨 아래 **`[프로젝트 이름]으로 이동(안전하지 않음)`** 클릭 (이 문구가 무섭게 보이지만, 본인이 복사한 파일이라 안전합니다.)
5.  `허용` 클릭

## 4. 오류 발생 시 대처법 (Troubleshooting)

갑자기 자동화가 작동하지 않을 때 다음 순서대로 확인해 주세요.

**증상 1: 버튼을 눌렀는데 아무 반응이 없거나 에러가 뜹니다.**
*   **원인:** 인터넷 연결 문제거나, 데이터 입력 형식이 틀렸을 수 있습니다.
*   **해결:**
    1.  입력한 데이터에 오타나 빈칸이 없는지 확인하세요. (특히 날짜 형식 등)
    2.  새로고침(F5) 후 다시 시도해 보세요.

**증상 2: 매일 자동으로 되던 것이 안 됩니다.**
*   **원인:** '트리거(자동 타이머)'가 풀렸을 수 있습니다.
*   **해결:**
    1.  구글 시트 상단 메뉴 `확장 프로그램` > `Apps Script` 클릭.
    2.  왼쪽 메뉴 중 `시계 아이콘(트리거)` 클릭.
    3.  목록에 아무것도 없다면 우측 하단 `트리거 추가`를 눌러 다시 설정해 주세요.

**증상 3: Netlify 사이트가 안 열립니다.**
*   **해결:** Netlify 계정에 로그인하여 `Deploys` 메뉴를 확인하세요. `Failed`라고 떠 있다면, 최근에 수정한 구글 시트 데이터가 웹사이트와 충돌한 것입니다. 최근 입력한 데이터를 지우고 다시 시도하세요.

이 시스템은 이제 대표님의 소유입니다. 원본이 훼손될까 걱정되신다면, 작업을 시작하기 전에 항상 **`파일 > 사본 만들기`**로 백업 파일을 하나 만들어두는 습관을 들이시면 가장 안전합니다.
