# 제미나이의 응답

반갑습니다. IT 기업 M&A 기술 실사 전문가이자 테크니컬 라이터로서, 귀하의 소중한 자동화 시스템이 구매자에게 안전하고 완벽하게 이관될 수 있도록 가이드를 작성해 드립니다.

기술을 잘 모르는 구매자도 쉽게 이해할 수 있도록 **'가게 인수'**에 비유하여 설명하고, 실무적인 매뉴얼과 체크리스트를 제공하겠습니다.

---

# 1. [개념 잡기] 공유 vs 소유권 이전
가장 먼저 구매자에게 이 개념을 설명해주셔야 나중에 분쟁이 생기지 않습니다.

*   **공유 (편집자 권한):** 가게의 **'보조 열쇠'**를 복사해 주는 것과 같습니다. 구매자가 문을 열고 들어와서 청소도 하고 물건도 채울 수 있지만, 원래 주인(판매자)이 마음이 바뀌어 자물쇠를 바꾸면 구매자는 더 이상 들어올 수 없습니다.
*   **소유권 이전:** 가게의 **'등기 권리증'**과 **'마스터키'**를 넘기는 것입니다. 이제 구매자가 진짜 주인이 되며, 오히려 판매자를 가게에서 내보낼 수 있는 권한을 갖게 됩니다. **우리는 이 단계까지 가야 합니다.**

---

# 2. 소유권 이전 및 권한 설정 체크리스트
구글 시스템은 보안이 철저하여, 단순히 파일만 넘긴다고 끝나는 것이 아닙니다. 특히 '자동화(스크립트)'는 주인이 바뀌면 작동을 멈출 수 있으니 아래 절차를 반드시 지켜주세요.

### ⚠️ 주의: 구글 계정 정책 (필독)
> **개인 Gmail(@gmail.com)**에서 만든 파일은 **기업용 계정(@company.com)**으로 소유권 이전이 **불가능**한 경우가 많습니다.
> *   **해결책:** 구매자에게 **개인 Gmail 계정**을 준비해달라고 하거나, 구매자가 파일을 '사본 만들기'로 복사해가도록 해야 합니다. (이 가이드는 **소유권 이전**을 기준으로 작성되었습니다.)

### ✅ 단계별 이관 체크리스트

**1단계: 구글 시트 및 스크립트 소유권 넘기기 (등기 이전)**
*   [ ] **파일 공유:** 이관할 구글 시트 3개 우측 상단 [공유] 버튼 클릭 -> 구매자 이메일 입력 -> '편집자'로 추가.
*   [ ] **소유권 이전:** 다시 [공유] 버튼 클릭 -> 구매자 이름 옆 권한 설정 클릭 -> **[소유권 이전]** 선택.
*   [ ] **초대 수락:** 구매자가 이메일로 온 '소유권 이전 요청'을 수락해야 완료됩니다. (구매자에게 꼭 수락하라고 말씀해 주세요.)
*   [ ] **판매자 권한 삭제:** (선택 사항) 구매자가 소유권을 받은 후, 판매자(나)의 계정을 목록에서 삭제하거나 '뷰어(보기 전용)'로 변경합니다.

**2단계: 앱스 스크립트(자동화 로봇) 재승인 (근로계약 갱신)**
> *비유: 가게 주인이 바뀌었으니, 일하는 점원(로봇)도 새 주인과 다시 근로 계약을 맺어야 일을 합니다.*
*   [ ] **스크립트 편집기 접속:** 구매자가 구글 시트 메뉴 중 [확장 프로그램] -> [Apps Script] 클릭.
*   [ ] **트리거(시계) 재설정:** (중요) 자동화가 '매일 아침 9시' 처럼 시간에 맞춰 도는 기능이라면, **기존 트리거를 삭제하고 구매자 계정으로 다시 설정**해야 합니다.
    *   좌측 시계 아이콘(트리거) 클릭 -> 기존 트리거 삭제 -> [트리거 추가] -> 기존과 동일한 조건으로 설정 -> **[저장] 시 나오는 팝업에서 '허용(Allow)' 클릭.**
*   [ ] **최초 실행:** 스크립트 상단 [실행] 버튼을 한 번 눌러, 권한 승인 팝업이 뜨면 [고급] -> [이동(안전하지 않음)] -> [허용]을 눌러줍니다.

**3단계: 외부 서비스 (네틀리파이 등) 계정 인계**
*   [ ] **계정 정보 변경:** 네틀리파이 로그인 -> [User Settings] -> 이메일 주소를 구매자의 이메일로 변경 -> 비밀번호 재설정 링크 발송.
*   [ ] **결제 정보 삭제:** 혹시 등록된 내 카드가 있다면 삭제합니다.

# 3. 시스템 인계인수 매뉴얼 (구매자용)

**문서 제목:** 자동화 시스템 운영 및 관리 매뉴얼
**작성 일자:** 202X. XX. XX
**수신:** [구매자 성함] 님

## 1. 시스템 개요 (이 가게는 어떻게 돌아가나요?)
이 시스템은 사장님(구매자)이 주무시는 동안에도 자동으로 데이터를 수집하고 정리해 줍니다.
*   **구글 시트 (3개):** 데이터가 쌓이는 장부입니다. (A시트: 원본, B시트: 가공, C시트: 결과)
*   **앱스 스크립트:** 장부를 정리하는 투명 로봇입니다.
*   **네틀리파이:** 결과 데이터를 고객들에게 보여주는 웹사이트(간판)입니다.

## 2. 데이터 수정 방법 (장부 관리법)
시스템이 고장 나지 않게 하려면 **'건드려도 되는 곳'**과 **'안 되는 곳'**을 구분해야 합니다.

*   **흰색 배경 셀:** 자유롭게 수정하고 입력해도 됩니다. (예: 날짜, 고객명, 금액 등)
*   **회색 배경 셀 (또는 수식이 있는 셀):** ⛔ **절대 수정 금지.** 이곳은 로봇이 계산하는 영역입니다. 지우면 자동화가 멈춥니다.
*   **행/열 삭제:** 데이터를 지울 때는 내용(글자)만 지우세요. 행이나 열 자체를 삭제하면 로봇이 길을 잃을 수 있습니다.

## 3. 오류 발생 시 대처법 (비상 조치)

**상황 A: "자동화가 안 도는 것 같아요." (데이터가 안 들어옴)**
1.  **권한 확인:** 구글 시트 메뉴 [확장 프로그램] -> [Apps Script]를 켭니다.
2.  **수동 실행:** 상단 메뉴바의 `Run`(실행) 버튼을 눌러보세요.
3.  **빨간 경고창이 뜬다면?**
    *   *"Authorization required"*: 권한이 풀린 것입니다. [권한 검토] -> 계정 선택 -> [고급] -> [이동] -> [허용]을 순서대로 누르세요.
    *   *"Quota exceeded"*: 구글의 하루 사용량을 초과했습니다. 내일 다시 실행되니 기다리시면 됩니다.

**상황 B: "웹사이트(네틀리파이) 내용이 안 바뀌어요."**
1.  구글 시트의 데이터가 정상적으로 저장되었는지 확인하세요.
2.  네틀리파이 사이트에 접속하여 [Deploys] 메뉴를 확인하세요. 'Failed'라고 떠 있다면 일시적인 서버 오류일 수 있으니 [Trigger deploy] 버튼을 눌러 재시도하세요.

## 4. 유지보수 및 주의사항
*   **구글 비밀번호 변경 시:** 구글 계정 비밀번호를 바꾸면 스크립트 연결이 끊길 수 있습니다. 위 '상황 A'의 방법대로 스크립트 권한을 다시 허용해 주세요.
*   **시트 이름 변경 금지:** 시트 하단의 탭 이름(예: 'Sheet1', 'Data')을 바꾸면 로봇이 장부를 못 찾습니다. 이름은 그대로 두세요.
