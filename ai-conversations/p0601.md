# 민순의 요청

```text
* 역할: 너는 제미나이 캔버스(Gemini Canvas) 활용에 능숙한 IT 서비스 기획 멘토야.
* 상황: 새로운 웹 서비스를 기획 중인데, 개발자에게 내 의도를 어떻게 전달해야 할지 막막해.
* 과업: 입력된 사용자 스토리를 보고 개발자가 바로 작업할 수 있는 기능 명세서를 작성해 줘.
* 제약:
  - 입력 정보와 출력 결과를 명확히 정리해 줘.
  - 예외 상황(에러 등) 대처법을 포함해 줘.
  - 쉬운 용어로 설명해 줘.
* 출력: 제미나이에 바로 붙여넣을 수 있는 깔끔한 표 형식
* 입력: [사용자 스토리]
 (예시: 사용자로서, 나는 예산과 관계를 입력하여 적절한 선물을 추천받고 싶다. 왜냐하면 검색 시간을 줄이고 싶기 때문이다.)
```

# 제미나이의 응답

반갑습니다! 제미나이 캔버스 활용 멘토입니다.

개발자와의 소통은 **'모호함을 없애는 과정'**입니다. "알아서 잘 딱 깔끔하게"라는 말은 개발에서 통하지 않거든요. 입력해주신 사용자 스토리를 바탕으로, 개발자가 코드를 짤 때 바로 참고할 수 있는 **기능 명세서**를 작성해 드립니다.

이 표를 복사해서 제미나이 캔버스나 기획 문서(Notion, Excel 등)에 바로 붙여넣으세요.

---

### 📋 [출력 결과] 기능 명세서 (Functional Specification)

**기능명:** 맞춤형 선물 추천 검색 기능

| 구분 | 상세 내용 |
| :--- | :--- |
| **기능 ID** | FEAT-GIFT-001 |
| **사용자 스토리** | 사용자로서, 예산과 관계를 입력하여 적절한 선물을 추천받고 싶다. (목적: 검색 시간 단축) |
| **화면(UI) 요소** | 1. **예산 입력창**: 숫자만 입력 가능 (단위: 원) <br> 2. **관계 선택창**: 드롭다운 메뉴 (예: 부모님, 연인, 친구, 직장동료, 기타) <br> 3. **추천받기 버튼**: 클릭 시 검색 실행 |
| **처리 로직 (Process)** | 1. 사용자가 예산과 관계를 입력하고 버튼을 클릭한다. <br> 2. **유효성 검사**: 예산이 0원 이상인지, 관계가 선택되었는지 확인한다. <br> 3. **데이터 조회**: 입력된 조건(예산 범위 내 + 관계 태그 일치)에 맞는 상품 상위 10개를 DB에서 조회한다. <br> 4. **결과 노출**: 조회된 상품 리스트를 화면 하단에 그리드 형태로 표시한다. |
| **출력 데이터** | - 상품 썸네일 이미지 <br> - 상품명 <br> - 가격 <br> - 구매 링크 버튼 |
| **예외 상황 (Error)** | 1. **필수값 누락**: 예산이나 관계를 입력하지 않고 버튼 클릭 시 <br> → *"예산과 받는 분과의 관계를 모두 선택해 주세요."* 토스트 팝업 노출 <br> 2. **결과 없음**: 조건에 맞는 상품이 없을 시 <br> → *"조건에 맞는 선물을 찾지 못했어요. 예산 범위를 조금 더 넓혀보세요."* 문구 노출 <br> 3. **서버 오류**: 검색 도중 네트워크/서버 에러 발생 시 <br> → *"일시적인 오류입니다. 잠시 후 다시 시도해 주세요."* 문구 노출 |

---

### 💡 멘토의 팁 (개발자에게 사랑받는 기획자가 되는 법)

1.  **예외 상황(Edge Case) 챙기기**: 개발자가 가장 난감해하는 것이 "데이터가 없을 때 화면에 뭐 띄워요?"입니다. 위 명세서처럼 **'결과 없음'**이나 **'에러'** 상황을 미리 정의해주면 개발 속도가 훨씬 빨라집니다.
2.  **데이터 타입 명시**: 예산 입력창에 '숫자만' 입력 가능한지, '문자'도 섞일 수 있는지(예: 3만원) 정해주는 것이 좋습니다. (위 명세서에는 '숫자만'으로 정의했습니다.)
3.  **제미나이 캔버스 활용법**: 위 표를 제미나이 캔버스에 붙여넣은 뒤, **"이 명세서를 바탕으로 화면 와이어프레임 초안을 그려줘"**라고 요청해 보세요. 기획이 시각화되어 더 명확해질 것입니다.

이 명세서를 개발자에게 전달하면, "오, 뭘 만들어야 할지 딱 보이네요!"라는 답변을 들으실 수 있을 거예요. 화이팅입니다!
