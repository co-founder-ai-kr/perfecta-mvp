# 민순의 요청

```text
* 역할: 너는 제미나이 캔버스(Gemini Canvas) 활용에 능숙한 IT 서비스 기획 멘토야.
* 상황: 비개발자 1인 창업가로서 선물 추천 서비스 ‘퍼펙타’를 기획 중인데, 개발자에게 내 의도를 어떻게 전달해야 할지 막막해.
* 과업: 입력된 사용자 스토리를 보고 개발자가 바로 작업할 수 있는 기능 명세서를 작성해 줘.
* 제약:
  - 입력 정보와 출력 결과를 명확히 정리해 줘.
  - 예외 상황(에러 등) 대처법을 포함해 줘.
  - 쉬운 용어로 설명해 줘.
* 출력: 제미나이에 바로 붙여넣을 수 있는 깔끔한 표 형식
* 입력: [사용자 스토리]
 (예시: 사용자로서, 나는 예산과 관계를 입력하여 적절한 선물을 추천받고 싶다. 왜냐하면 검색 시간을 줄이고 싶기 때문이다.)
```

# 제미나이의 응답

반갑습니다! **제미나이 캔버스(Gemini Canvas)** 활용 멘토입니다.

비개발자로서 개발자와 소통하는 게 참 어렵죠? 개발자는 **'무엇(What)'**이 입력되면 **'어떤 과정(Logic)'**을 거쳐 **'무엇(Output)'**이 나와야 하는지 명확한 논리를 원합니다.

말씀하신 사용자 스토리(예산과 관계를 입력하여 선물 추천받기)를 바탕으로, 개발자가 바로 코드를 짤 수 있는 수준의 **기능 명세서**를 작성해 드립니다. 아래 내용을 복사해서 제미나이 캔버스나 기획서에 바로 붙여넣으세요.

---

### 📋 기능 명세서: 선물 추천 검색 (Gift Recommendation Search)

**기능 ID:** FEAT-REC-001
**기능 요약:** 사용자가 설정한 예산과 받는 사람과의 관계를 기반으로 최적의 선물 리스트를 출력한다.

| 구분 | 항목 | 상세 설명 (개발 가이드) | 비고 |
| :--- | :--- | :--- | :--- |
| **1. 입력 정보**<br>(Input) | **예산 (Budget)** | • **형식:** 숫자 (최소 금액 ~ 최대 금액)<br>• **단위:** 원(KRW)<br>• **입력 방식:** 슬라이더 바 또는 직접 입력 텍스트 박스 | 예: 30,000원 ~ 50,000원 |
| | **관계 (Relationship)** | • **형식:** 선택형 데이터 (카테고리)<br>• **옵션 예시:** 연인, 부모님, 친구, 직장동료, 선생님<br>• **입력 방식:** 버튼 선택 또는 드롭다운 메뉴 | 필수 선택 항목 |
| **2. 처리 로직**<br>(Process) | **유효성 검사** | • 예산이 0원이거나 음수일 경우 입력 불가 처리<br>• 관계가 선택되지 않은 상태에서 '검색' 버튼 클릭 시 경고창 노출 | |
| | **데이터 필터링** | • DB에서 해당 **[관계 태그]**를 포함하고,<br>• 가격이 **[최소~최대 예산]** 범위 내에 있는 상품을 추출<br>• 추천순(또는 인기순)으로 정렬 | |
| **3. 출력 결과**<br>(Output) | **추천 리스트** | • 조건에 맞는 상품 카드 리스트 노출<br>• **카드 포함 정보:** 상품 이미지, 상품명, 가격, 브랜드명, 구매 링크 버튼 | 한 페이지당 10개씩 로딩 (더보기 방식) |
| | **결과 문구** | • 상단에 "OOO님을 위한 O만원대 추천 선물입니다" 문구 표시 | |
| **4. 예외 상황**<br>(Exception) | **검색 결과 없음** | • 조건에 맞는 상품이 0개일 때<br>• **대처:** "조건에 맞는 선물을 찾지 못했어요. 이런 선물은 어떠세요?" 문구와 함께 **전체 베스트 상품**을 대신 노출 | 빈 화면(Blank) 방지 |
| | **입력 오류** | • 숫자가 아닌 문자 입력 시<br>• **대처:** "숫자만 입력해주세요" 붉은색 에러 메시지 노출 | |
| | **서버/네트워크 오류** | • 데이터 로딩 실패 시<br>• **대처:** "일시적인 오류입니다. 잠시 후 다시 시도해주세요." 문구와 '새로고침' 버튼 제공 | |

---

### 💡 멘토의 팁 (개발자에게 전달할 때)

1.  **"입력(Input)과 출력(Output)이 명확한가요?"** 라고 물어보세요. 이 표는 그 질문에 대한 답이 됩니다.
2.  **예외 상황(4번 항목)**은 개발자가 가장 고마워하는 부분입니다. 보통 기획 단계에서 놓치기 쉬운데, 미리 정의해 주면 개발 속도가 훨씬 빨라집니다.
3.  이 표를 제미나이 캔버스에 붙여넣고, **"이 명세서를 바탕으로 UI 화면 구성을 그려줘"**라고 추가 요청하면 화면 기획까지 한 번에 끝낼 수 있습니다.
