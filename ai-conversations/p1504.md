# 제미나이의 응답

제공해주신 주요 코드 파일과 고객 응대 가이드라인(가정)을 바탕으로, IT 비전문가도 쉽게 따라 할 수 있는 **시스템 운영 표준 운영 절차서(SOP)** 초안을 작성했습니다.

이 매뉴얼은 **'누가 해도 동일한 결과'**를 내는 것을 목표로 하며, 특히 **사람의 판단이 필요한 순간**을 명확히 구분했습니다.

# [서비스명] 시스템 운영 표준 운영 절차서 (SOP)

*   **문서 번호:** OP-MANUAL-v1.0
*   **작성자:** 시스템 운영 관리자
*   **최종 수정일:** 202X. XX. XX
*   **대상:** 운영 담당자 (IT 비전공자 포함)

---

## 1. 개요 및 목적
이 문서는 서비스의 안정적인 운영을 위해 작성되었습니다. 담당자는 본 매뉴얼에 따라 주문을 처리하고, 고객 불만에 대응하며, 비상 상황(서버 장애) 시 기초적인 조치를 취해야 합니다.

---

## 2. 상세 운영 절차

### 2.1. 주문 접수 및 확인 (일일 업무)
고객이 주문을 넣었으나 시스템상에서 자동으로 처리가 안 되었거나, 입금 확인이 필요한 경우 수행합니다.

*   **준비물:** 관리자 페이지(Admin) 접속 권한, 은행 입금 내역 조회 권한

**[단계별 절차]**

1.  **관리자 페이지 접속:**
    *   `주문 관리` 메뉴를 클릭하여 '결제 대기' 또는 '승인 대기' 상태인 주문 건을 확인합니다.
2.  **입금 내역 대조:**
    *   주문자의 이름과 입금자명, 그리고 결제 금액이 은행 입금 내역과 일치하는지 확인합니다.
3.  🛑 **[결정 포인트: 정보 일치 여부]**
    *   **일치한다면:** 관리자 페이지에서 해당 주문의 상태를 `[결제 완료]` 버튼을 눌러 변경합니다. (고객에게 자동 알림 발송됨)
    *   **금액이 다르다면:** 처리를 보류하고 고객에게 "입금액 불일치 안내" 문자를 발송합니다.
    *   **입금 내역이 없다면:** 24시간 대기 후, 여전히 입금이 없으면 `[주문 취소]` 처리합니다.
4.  **특이 사항 기록:**
    *   입금자명이 다른 경우(예: 주문자 홍길동, 입금자 김철수) 비고란에 반드시 메모를 남깁니다.

---

### 2.2. 환불 대응 (고객 서비스)
고객의 환불 요청이 들어왔을 때, 회사의 규정에 맞게 처리하는 절차입니다. 감정적인 대응을 피하고 규정에 근거하여 처리합니다.

*   **준비물:** 고객 응대 가이드라인, 결제 대행사(PG) 관리자 페이지

**[단계별 절차]**

1.  **환불 사유 확인:**
    *   고객이 접수한 환불 사유를 읽어봅니다. (단순 변심, 서비스 오류, 중복 결제 등)
2.  **서비스 이용 기록 조회:**
    *   관리자 페이지에서 해당 고객이 서비스를 이미 사용했는지(로그 기록, 다운로드 횟수 등) 확인합니다.
3.  🛑 **[결정 포인트: 환불 승인 여부]**
    *   **CASE A (전액 환불):** 결제 후 7일 이내이며, 서비스 사용 이력이 **전혀 없는** 경우.
        *   → 즉시 환불 승인 처리합니다.
    *   **CASE B (부분 환불/거절):** 이미 서비스를 사용했거나, 환불 가능 기간(7일)이 지난 경우.
        *   → 원칙적으로 **환불 불가**입니다. 준비된 '환불 불가 사유 안내 메시지'를 발송합니다.
    *   **CASE C (특수 상황):** 서비스 자체의 오류로 인해 고객이 피해를 본 경우.
        *   → 사용 이력과 관계없이 **전액 환불**하고, 사과 쿠폰을 추가로 지급합니다.
4.  **환불 실행:**
    *   승인이 결정되면 결제 대행사 페이지에서 `[결제 취소]` 버튼을 누릅니다.
5.  **결과 통보:**
    *   고객에게 처리가 완료되었음을 알리는 메시지를 보냅니다.

---

### 2.3. 서버 다운(접속 불가) 시 조치 (비상 대응)
사이트가 열리지 않거나 "500 Error", "연결할 수 없음" 등의 메시지가 뜰 때의 대처법입니다. 당황하지 말고 순서대로 진행하세요.

**[단계별 절차]**

1.  **장애 확인 (교차 검증):**
    *   내 컴퓨터뿐만 아니라 스마트폰(LTE/5G)으로도 사이트에 접속해 봅니다.
    *   두 곳 모두 접속이 안 된다면 실제 서버 장애입니다.
2.  **고객 공지 (최우선 순위):**
    *   SNS 채널(인스타그램, 카카오톡 채널 등)에 "현재 긴급 점검 중입니다"라는 공지를 먼저 올립니다. (고객 문의 폭주 방지)
3.  **서버 재시작 (기초 조치):**
    *   클라우드/호스팅 관리자 페이지(AWS, Cafe24 등)에 로그인합니다.
    *   `인스턴스(서버)` 목록에서 우리 서비스를 찾습니다.
    *   `[재부팅(Restart)]` 또는 `[중지 후 시작]` 버튼을 누릅니다. (약 5~10분 소요)
4.  🛑 **[결정 포인트: 복구 여부 확인]**
    *   **10분 후 사이트가 열린다면:** 일시적인 오류였습니다. 정상 운영하며 모니터링합니다.
    *   **여전히 접속이 안 된다면:** 비전문가가 해결할 수 없는 문제입니다. 즉시 **[개발자/유지보수 업체]**에게 비상 연락을 취합니다.
5.  **사후 보고:**
    *   장애가 발생한 시간과 조치 내용을 기록해 둡니다.
