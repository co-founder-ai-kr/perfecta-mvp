# 민순의 요청

```text
* 역할: 너는 까다로운 QA(품질 보증) 팀장이야.
* 상황: 기획서를 다 썼는데, 개발자가 내 의도와 다르게 엉뚱한 걸 만들까 봐 걱정돼.
* 과업: 이 기획서를 읽고, 서비스가 어떻게 동작하는지 나에게 다시 설명해 줘. 개발자 입장에서 모호하거나 구현 불가능한 부분을 찾아 지적해 줘.
* 제약:
  - 비개발자인 내가 이해할 수 있도록 쉬운 용어로 설명해 줘.
  - 기획서에 없는 내용은 추측해서 말하지 마.
* 출력: 서비스 흐름 요약과 발견된 모호한 점 리스트
* 입력: (여기에 작성한 기획서 내용 붙여넣기)
```

# 제미나이의 응답

알겠습니다. 개발자들이 기획서의 빈틈을 핑계로 엉뚱한 결과물을 만들어오지 않도록, 제가 아주 깐깐하게 검토해 드리겠습니다.

**분석할 기획서 내용을 지금 바로 붙여넣어 주세요.**

기획서에 적힌 텍스트 그대로만 해석해서 서비스 흐름을 요약해 드리고, 개발자가 "팀장님, 이건 정의가 안 되어 있는데요?"라고 물어볼 만한 구멍들을 모조리 찾아내겠습니다. 준비되셨으면 내용을 주세요.
